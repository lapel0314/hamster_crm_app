# 로컬 안전장치 체크리스트

랭가 지시 기준: 보안 기능, 로그인, 권한 관리는 만들지 않는다. 개인 PC에서 혼자 쓰는 로컬 오프라인 Windows 데스크톱 앱이다.

## 범위
- 인증/로그인 없음
- 사용자 권한 분리 없음
- 서버/클라우드 동기화 없음
- 과한 보안 설계 없음

## 필요한 최소 안전장치

### 1. 로컬 DB 손상 방지
- SQLite 사용 시 WAL 모드 권장.
- 앱 종료 중 저장 작업이 끊기지 않도록 저장 트랜잭션 사용.
- DB 파일 위치는 설치 폴더가 아니라 사용자 데이터 폴더 사용.
- 앱 시작 시 DB 파일 존재/열림 여부 확인.

권장 경로:
- `%APPDATA%\GoldenHamsterCRM\hamster_crm.db`

### 2. 자동 백업
- 앱 실행 시 하루 1회 자동 백업.
- 백업 파일명은 날짜 포함.
- 최근 30개 백업만 보관.

권장 경로:
- `%APPDATA%\GoldenHamsterCRM\backups\hamster_crm_YYYY-MM-DD.db`

### 3. 수동 백업/복원
- 설정 또는 대시보드 하단에 “백업하기”, “복원하기” 버튼 제공.
- 복원 전 현재 DB 자동 백업.
- 복원 후 앱 재시작 안내.

### 4. 실수 삭제 방지
- 고객DB/가망고객 삭제는 즉시 삭제하지 않고 확인 팝업 표시.
- 가능하면 `deleted_at` 필드로 소프트 삭제.
- “최근 삭제 항목 복구” 기능은 2차 기능으로 고려.

### 5. 설치/업데이트 시 데이터 보존
- Inno Setup 설치 경로와 데이터 경로를 분리.
- 언인스톨 시 사용자 데이터 삭제하지 않음.
- 업데이트 설치 시 `%APPDATA%\GoldenHamsterCRM` 유지.

### 6. 개인정보 노출 최소화
- 로그인은 없지만 화면에서 휴대폰번호 전체 표시 여부는 선택 가능하게 고려.
- 기본은 전체 표시, 필요 시 검색/표에서 중간 마스킹 옵션 추가 가능.

### 7. 엑셀 내보내기 주의
- 고객DB/가망고객 내보내기 기능을 만들 경우 저장 위치 선택 팝업 제공.
- 내보내기 완료 후 파일 경로 표시.

## 개발 우선순위
1. DB 파일 사용자 데이터 폴더 저장
2. 앱 실행 시 자동 백업
3. 삭제 확인 팝업
4. Inno 업데이트 시 데이터 보존
5. 수동 백업/복원
