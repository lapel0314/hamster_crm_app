# 햄스터 CRM 리서치 노트

## 프로젝트 전제
- 햄스터 분양 업체 전용 신규 프로그램.
- 서버 없는 로컬 오프라인 Windows 데스크톱 앱.
- 로그인 없음.
- 개인 PC 1인 사용.
- 디자인 우선: 골든햄스터, 아기자기한 UI.
- 설치파일: Inno Setup.

## 현재 확정 페이지
1. 고객등록
2. 고객DB
3. 가망고객
4. 대시보드

## 기본 업무 흐름

### 1. 가망고객 상담
- 고객이 문의한다.
- 상담날짜, 방문예정날짜, 고객명, 성별, 휴대폰번호, 관심 분양/구매 항목, 예상 매출/원가, 메모를 기록한다.
- 방문 예정 고객은 가망고객 화면에서 날짜 기준으로 확인한다.

### 2. 방문/분양 확정
- 방문 후 실제 분양 또는 구매가 발생하면 고객DB에 등록한다.
- 가망고객에서 고객DB로 전환하는 기능은 2차 기능으로 고려한다.

### 3. 고객DB 관리
- 실제 고객 기록을 날짜별로 조회한다.
- 고객명/휴대폰번호/분양/구매/메모로 검색한다.
- 날짜 필터로 기간별 고객을 본다.

### 4. 대시보드 확인
- 월별 분양 추이와 구매 추이를 확인한다.
- 1월~12월 매출/원가를 확인한다.
- 이번달 어떤 분양/구매 항목이 많이 나갔는지 순위를 본다.

## 누락 가능성이 있는 필드 후보
현재 랭가가 지정한 필드를 우선으로 한다. 아래는 추후 추가 후보이며, 1차 MVP에는 넣지 않아도 된다.

### 고객 관련
- 유입경로: 블로그, 인스타, 지인소개, 당근, 네이버 등
- 지역
- 재방문 여부
- 상담 상태: 상담중, 방문예정, 보류, 완료, 취소

### 햄스터 분양 관련
- 햄스터 종류: 골든, 푸딩, 로보, 드워프 등
- 개체명/번호
- 생년월일 또는 주령
- 분양가
- 분양 상태: 가능, 예약, 완료

### 구매 관련
- 구매 품목 세분화: 사료, 케이지, 베딩, 장난감 등
- 수량
- 결제수단: 현금, 카드, 계좌이체

### 사후관리 관련
- 사후 연락일
- 특이사항
- 재구매 메모

## 1차 MVP 범위 권장
랭가가 지정한 페이지/필드만 먼저 구현한다.

### MVP 포함
- 고객등록
- 고객DB 목록/검색/날짜 필터
- 가망고객 목록/검색/상담날짜/방문예정날짜 필터
- 고객/가망고객 상세 보기 및 수정
- 삭제 확인 팝업과 소프트 삭제
- 대시보드 월별 그래프
- 월별 매출/원가 정산
- 이번달 분양/구매 순위
- 로컬 SQLite 저장
- 자동 백업 최소 기능
- 데이터 폴더 위치 안내 또는 백업 위치 표시

### MVP 제외
- 로그인/권한
- 서버/클라우드 동기화
- 직원 관리
- 문자/카카오 발송
- 영수증/계약서 자동 생성
- 햄스터 개체별 재고 관리
- 엑셀 가져오기/내보내기

## 2차 기능 후보
1. 가망고객 → 고객DB 전환 버튼
2. 엑셀 내보내기
3. 햄스터 개체 관리
4. 분양 계약서/안내문 출력
5. 자동 백업/복원 UI 강화
6. 빈 상태 일러스트/삭제 복구함
7. 월별 순이익 표시
8. 품목 자동완성/드롭다운 관리
9. 상담 상태 칸반 또는 상태별 필터

## MVP 우선순위 제안

### P0: 반드시 먼저 동작해야 하는 핵심
- 고객등록 저장/수정/소프트삭제.
- 고객DB 목록, 검색, 날짜 필터.
- 가망고객 등록, 목록, 상담날짜/방문예정날짜 필터.
- SQLite 로컬 저장과 앱 재실행 후 데이터 유지.
- 기본 골든햄스터 컨셉 UI: 따뜻한 베이지/골드 톤, 둥근 카드, 귀여운 빈 상태.

### P1: MVP 완성도를 올리는 기능
- 대시보드 월별 분양/구매 추이.
- 월별 매출/원가 합계.
- 이번달 분양/구매 순위.
- 자동 백업 최소 기능과 백업 위치 표시.
- 입력값 검증: 날짜, 휴대폰번호, 매출/원가 숫자.

### P2: 출시 직전 polish
- 삭제 확인/복구 UX 강화.
- 빈 목록 일러스트와 안내 문구.
- 창 크기 변경 대응, 작은 화면 스크롤 확인.
- 설치파일 아이콘/앱 이름/시작 메뉴 바로가기 확인.

## 추가 운영 흐름 제안

### 하루 운영 흐름
1. 오전: 대시보드에서 이번달 실적과 오늘/이번주 방문예정 고객 확인.
2. 문의 발생: 가망고객에 상담 내용과 방문예정날짜 입력.
3. 방문 완료: 실제 분양/구매가 있으면 고객등록으로 고객DB에 확정 기록.
4. 마감: 고객DB와 대시보드 금액 확인, 자동 백업 상태 확인.

### 데이터 수정/삭제 흐름
- 잘못 입력한 고객은 목록에서 상세 화면으로 들어가 수정한다.
- 삭제는 즉시 영구 삭제하지 않고 `삭제됨` 상태로 숨긴다.
- 복구 UI는 2차로 미뤄도 되지만, DB 구조는 소프트 삭제를 전제로 잡는 것이 안전하다.

### 백업/복구 흐름
- 앱 종료 시 또는 하루 1회 자동 백업을 만든다.
- 백업 파일은 날짜가 포함된 파일명으로 저장한다.
- MVP에서는 복구 버튼이 없어도 백업 위치를 사용자에게 보여준다.
- 데이터 손실 방지를 위해 최근 N개 백업만 보관하는 정책을 정한다.

## 추가 요구사항 제안

### 로컬/오프라인 요구사항
- 인터넷 연결 없이 실행, 등록, 조회, 대시보드가 모두 동작해야 한다.
- 외부 서버, 계정, 로그인, 클라우드 동기화는 사용하지 않는다.
- 앱 데이터는 사용자 PC 로컬 폴더에 저장한다.
- 설치/삭제 후 데이터 보존 정책을 정해야 한다. 기본 권장: 프로그램 삭제와 데이터 삭제는 분리.

### 입력/검증 요구사항
- 고객명 또는 휴대폰번호 중 최소 하나는 필수.
- 상담날짜/등록날짜는 기본값을 오늘로 자동 입력.
- 방문예정날짜는 선택 입력으로 둔다.
- 매출/원가는 빈 값이면 0으로 집계한다.
- 휴대폰번호는 하이픈 유무와 관계없이 검색되도록 정규화한다.

### UI/디자인 요구사항
- 골든햄스터 컨셉을 앱 전체에 일관 적용한다.
- 색상: 크림/베이지 배경, 골드 포인트, 브라운 텍스트 계열 권장.
- 카드형 레이아웃, 둥근 버튼, 귀여운 아이콘/일러스트 사용.
- 업무용 프로그램이므로 귀여워도 가독성과 입력 속도를 해치지 않는다.
- 빈 상태, 저장 성공, 삭제 확인 등 상태 메시지도 컨셉에 맞춘다.

### 설치/배포 요구사항
- Windows 데스크톱 단일 설치파일을 목표로 한다.
- Inno Setup 설치 시 바탕화면/시작메뉴 바로가기 옵션을 제공한다.
- 앱 이름, 아이콘, 설치 경로, 데이터 저장 경로를 사전에 확정한다.
- 기존 `crm_app`은 수정하지 않고 신규 프로젝트로 분리한다.

## 데이터 집계 기준 제안

### 월별 분양 추이
- 고객DB의 `분양` 필드가 비어 있지 않은 건수를 월별 집계.
- 같은 텍스트 기준으로 순위 계산.

### 월별 구매 추이
- 고객DB의 `구매` 필드가 비어 있지 않은 건수를 월별 집계.
- 같은 텍스트 기준으로 순위 계산.

### 정산현황
- 총 매출: 해당 월 `매출` 합계.
- 총 원가: 해당 월 `원가` 합계.
- 순이익: 총 매출 - 총 원가. UI에 표시할지는 랭가 확인 필요.

## 확인 필요 사항
1. `분양`과 `구매`는 자유입력으로 할지, 드롭다운으로 할지.
2. 매출/원가 단위는 원화 정수만 사용할지.
3. 고객DB에서 수정/삭제 기능을 1차에 넣을지.
4. 가망고객을 고객DB로 전환하는 버튼을 1차에 넣을지.
5. 대시보드 순위는 텍스트 동일값 기준으로 집계해도 되는지.
6. 고객DB와 가망고객을 별도 테이블로 둘지, 상태값으로 한 테이블에서 관리할지.
7. 자동 백업 주기와 보관 개수: 앱 종료 시/하루 1회, 최근 7개/30개 등.
8. 프로그램 삭제 시 로컬 데이터도 삭제할지, 보존할지.
9. 대시보드 기준 날짜는 등록일, 분양일, 구매일 중 무엇으로 할지.
10. 앱 이름과 아이콘 문구를 확정할지: 예) 골든햄 CRM, 햄찌고객노트.

## 권장 결정
- 1차는 자유입력으로 빠르게 만든다.
- 이후 데이터가 쌓이면 드롭다운/자동완성으로 개선한다.
- 삭제는 실수 방지를 위해 확인 팝업과 소프트 삭제로 처리한다.
