# 햄스터 CRM 요구사항

## 개발 방향
- 기존 CRM과 별개인 신규 프로그램.
- 서버 없음: 로컬 오프라인 전용.
- 대상: Windows 데스크톱 앱.
- 로그인 없음: 개인 컴퓨터에서 1인 사용.
- 보안 기능은 최소화. 단, 로컬 데이터 파일 손상 방지와 백업은 개발 관점에서 고려.
- 설치파일: Inno Setup 사용.
- 디자인 중요도 높음.
- 디자인 컨셉: 골든햄스터, 따뜻한 크림/골드 톤, 아기자기한 버튼, 햄스터 이미지/일러스트 포함.

## 페이지
1. 고객등록
2. 고객DB
3. 가망고객
4. 대시보드

## 고객등록
입력 필드:
- 날짜
- 고객명
- 성별
- 휴대폰번호
- 분양
- 구매
- 매출
- 원가
- 메모

## 고객DB
표시 필드:
- 날짜
- 고객명
- 성별
- 휴대폰번호
- 분양
- 구매
- 매출
- 원가
- 메모

기능:
- 날짜 필터: 달력 형식
- 검색 기능

## 가망고객
표시/입력 필드:
- 상담날짜
- 방문예정날짜
- 고객명
- 성별
- 휴대폰번호
- 분양
- 구매
- 매출
- 원가
- 메모

기능:
- 상담날짜 필터: 달력 형식
- 방문예정날짜 필터: 달력 형식
- 검색 기능

## 대시보드
그래프:
- 월별 분양 추이
- 월별 구매 추이

정산현황:
- 1월~12월 각각 총 매출
- 1월~12월 각각 총 원가

순위:
- 이번달 분양 순위
- 이번달 구매 순위

## 1차 기술 가정
- Windows 데스크톱: Flutter Windows 또는 Electron 후보.
- 로컬 저장소: SQLite 권장.
- 설치파일: Inno Setup.
- 디자인 산출: Figma 플러그인으로 와이어프레임/시안 자동 생성.
